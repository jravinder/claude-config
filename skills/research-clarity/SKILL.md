# Research Clarity

When building or reviewing pages for Biotech Research Studio or HD Research Hub, apply these rules to every piece of content.

## 1. Attribution: Who said it?

Every data point on the page must be labeled with its source:

| Label | Meaning | Example |
|-------|---------|---------|
| **Published research** | Finding from a peer-reviewed paper | "MSH3 knockdown reduces expansion by 78%" |
| **AI-generated hypothesis** | Gemma synthesized this from multiple papers | "Combining FAN1 upregulation with MSH3 inhibition may be synergistic" |
| **Platform output** | The tool produced this (confidence scores, rankings) | "Confidence: 90/100" |
| **Human decision** | A person chose this (which targets, which papers, which prompts) | "5 targets selected based on GWAS evidence" |

When writing HTML or copy:
- Add `title` attributes or inline labels: `<span title="AI-generated by Gemma 4 from 75 papers">42 drug candidates</span>`
- Use small attribution text below data: `<div class="text-[10px] text-zinc-400">Source: Gemma 4 synthesis of 75 published papers</div>`
- Never present AI output as established fact. Always say "AI-identified" or "AI-generated" not "discovered" or "found."

## 2. Jargon: Make every term understandable

If a term wouldn't be understood by someone with no biology background, explain it inline or with a title attribute.

| Term | Plain explanation |
|------|------------------|
| Somatic CAG expansion | A DNA mutation that grows worse over time in brain cells |
| Drug repurposing | Testing existing approved drugs for a new disease |
| Protein target | A specific molecule in the body that a drug could interact with |
| AlphaFold | Google's AI that predicts the 3D shape of proteins |
| Clinical trial | A study testing a treatment in real patients |
| Preclinical | Tested in lab or animals, not yet in humans |
| ASO (antisense oligonucleotide) | A short piece of synthetic DNA that can turn off a gene |
| siRNA | A molecule that silences a specific gene |
| Splice modifier | A drug that changes how a gene's instructions are read |
| MutSbeta | A protein complex that recognizes DNA errors (drives expansion in HD) |
| GWAS | Genome-wide study that finds genetic variants linked to disease traits |

When writing HTML:
- Use `title` attributes: `<span title="A DNA mutation that grows worse over time in brain cells">somatic CAG expansion</span>`
- Or use inline parenthetical: "somatic CAG expansion (a DNA mutation that worsens over time)"
- Don't over-explain. One short phrase is enough.

## 3. Citation: Link to proof

Every claim should link to its source:

- **Numbers** (75 papers, 42 candidates) → link to the experiment page or raw data
- **Data sources** (PubMed, ClinicalTrials.gov) → link to the actual source
- **Tools used** (Gemma, AlphaFold, Ollama) → link to the tool's homepage
- **Code** → link to the specific file on GitHub
- **Papers referenced** → link to PubMed URL

Format: use amber-colored inline links, not footnotes.
```html
<a href="URL" target="_blank" rel="noopener noreferrer" class="text-amber-600 hover:underline">linked text</a>
```

## 4. Process: GitHub-first workflow

All changes to either repo should follow this workflow:

1. **Issue first** - Create a GitHub issue describing what needs to change and why
2. **Branch** - Work on a feature branch, not main
3. **PR with context** - PR description explains the change, links to the issue
4. **Review** - Codex or human reviews before merge
5. **Docs** - Update CLAUDE.md or relevant docs if the change affects how the platform works

For the hackathon-playbook repo (private), issues track strategy and competitive analysis.
For hd-research-agent and biotech-research-hub repos (public), issues track features and bugs.

## 5. When to apply this skill

Apply these rules whenever:
- Writing or editing HTML pages for either site
- Writing experiment reports or results pages
- Updating the landing page copy
- Creating new content that references data or research
- Building the Kaggle notebook or hackathon writeup
